2.2. Ejecución de código Python en ficheros planos¶

Una de las alternativas que podemos utilizar para ejecutar cdigo Python sin utilizar jupyter es escribir el cdigo en ficheros planos, que se ejecutarn directamente con el intrprete de Python.

Los ficheros planos de Python son archivos de texto que contienen cdigo fuente escrito en Python y que tienen la extensin .py. Estos archivos contendran el cdigo que hemos estado programando en celdas de cdigo en los notebooks. Del mismo modo que el cdigo en los notebooks, el cdigo Python en texto plano puede contener comentarios (ya sean de una sola lnea, comenzados con #; o multilnea, delimitados con ''') y estar indentado siguiendo la sintaxis de Python.

Para editar estos archivos, utilizaremos un editor de texto plano, o bien un entorno de desarrollo integrado (en ingls, se conoce como IDE, de integrated development environment).

Cuando estos archivos contienen un fragmento de cdigo pensado para ser ejecutado solo de manera directa por el usuario normalmente lo llamamos script de Python; mientras que si el cdigo est pensado para poderse importar desde otro archivo de Python lo llamamos mdulo.

En la carpeta python_code hay un ejemplo de mdulo en Python muy simple (hello_world.py), que est pensado para poderse importar, pero tambin para ejecutarse directamente.

El contenido del archivo es el siguiente:

#!/usr/bin/ python
# -*- coding: utf-8 -*-


def to_upper(x):
    return x.upper()


if __name__ == "__main__":
    print(to_upper("Hello world!"))

Notad cmo el archivo contiene nicamente cdigo (y comentarios de cdigo), pero no texto en markdown ni imgenes como podemos tener en los notebooks.

Por otra parte, el cdigo tiene dos partes diferenciadas:

  • La definicin de la funcin to_upper.
  • Un print de un mensaje.

El print se encuentra dentro de un bloque if, que comprueba si la variable __name__ es igual a la cadena de caracteres __main__. La variable __name__ es una variable especial asignada por el intrprete de Python, que toma el valor __main__ cuando el archivo se ejecuta como programa principal. En cambio, cuando se importa como mdulo, la variable toma el nombre del mdulo. Cuando se programan mdulos que pueden ser utilizados importados desde otros programas pero tambin como programas principales, es habitual utilizar esta construccin para indicar qu debe hacer el programa cuando se ejecuta como programa principal.

Por lo tanto, si ejecutamos este cdigo como programa principal, se mostrar el mensaje HELLO WORLD! por pantalla, mientras que si lo importamos como mdulo desde otro programa, este mensaje no se mostrar. En cuanto a la definicin de la funcin to_upper, esta siempre se ejecutar, ya que no se encuentra dentro del bloque if.

Para ejecutar este cdigo Python como programa principal, nos situaremos en la carpeta desde un terminal y ejecutaremos la siguiente instruccin:

$ python3 hello_world.py

Esto ejecutar el cdigo, que mostrar por pantalla el mensaje HELLO WORLD!.

Por otro lado, podemos importar este archivo como mdulo dentro de nuestro cdigo Python usando la instruccin import (como ya hemos estado haciendo con las libreras que hemos ido utilizando en la asignatura).

2.4. Gestión de librerías¶

Python es un lenguaje de programacin que dispone de muchos paquetes y mdulos externos (es decir, que no se encuentran en la librera estndar de Python). Para instalar un paquete, simplemente hay que ejecutar:

Esto har que se instale el paquete especificado, as como todas las dependencias que necesite para su ejecucin.

instala la versin 0.25.3 de pandas.

pip descarga los paquetes que queremos instalar del Python Package Index (PyPI), el ndice de paquetes por defecto de la comunidad Python, que se encuentra abierto a todos los desarrolladores. Adicionalmente, se puede usar pip para instalar paquetes de otros ndices, especificando la url con el flag --index-url.

2.4.1. Archivo de requisitos¶

El uso de libreras externas en Python facilita el desarrollo de programas (ya que los programadores pueden reutilizar cdigo ya existente), pero al mismo tiempo puede dificultar la portabilidad de cdigo entre diferentes mquinas (ya que un cdigo que dependa de libreras externas no funcionar si estas libreras no se encuentran instaladas en el sistema). Adems, a veces no solo ser necesario que las libreras estn instaladas, sino que ser necesario que la versin instalada sea la misma para la que se desarroll el cdigo (ya que diferentes versiones de una misma librera pueden disponer de funciones diferentes, tener distintos comportamientos por defecto, etc.).

Para ayudar a reproducir el entorno en el que un programa puede ejecutarse correctamente, en los proyectos de programacin en Python se suele crear un archivo (llamado habitualmente requirements.txt) que contiene un listado de todas las dependencias del proyecto, es decir, de todas las libreras que hay que tener instaladas para ejecutar el cdigo del proyecto. Adicionalmente, este archivo tambin puede especificar requisitos sobre las versiones de las libreras que hay que instalar, por ejemplo, indicando una versin concreta de una librera o bien la versin mnima que hay que tener.

A partir de un archivo de requisitos, podemos utilizar pip para instalar todas las libreras necesarias para un proyecto usando el comando siguiente:

$ pip install -r requirements.txt

Adems de gestionar la instalacin de paquetes, pip tambin permite generar un fichero con todas las libreras instaladas en un entorno de Python:

$ pip freeze requirements.txt

Adems de la gestin de libreras que permita asegurar la portabilidad de nuestro cdigo, a veces los programadores de Python tambin se pueden encontrar con conflictos en cuanto a las necesidades de los diversos proyectos en los que trabajan.   • Facilita gestionar dependencias (y evitar conflictos como los que hemos mencionado anteriormente).
  • Posibilita instalar libreras sin permiso de superusuario.
  • Asegura que no modificamos las libreras de Python del sistema.
  • Permite la reproduccin de los entornos de trabajo de los proyectos en otras mquinas.

Hay varias maneras de gestionar entornos virtuales en Python. En esta unidad didctica presentaremos la herramienta virtualenv.

def is_dotted_qad(value):
    """
    Checks whether a given value is in dotted quad format or nor:

    Dotted quad values are strings of four u1 integer separated
        by dotes:

    e.g. 127.0.0.1

    Args:
        value (:obj:`str`): the value to be checked.

    Returns:
        :obj:`bool`: True if the value is in dotted quad format.
            False otherwise.
    """

    if not isinstance(value, str):
        return False
    else:
        digits = value.split('.')
        if len(digits) != 4:
            return False
        else:
            if "/" in digits[3]:
                # Remove the mask from the IP (/value)
                digits[3] = digits[3][:digits[3].index('/')]
            return all(map(lambda x: x.isdigit(), digits))

def is_u1_string(value):
    """
    Checks whether a value is a unsigned 1-byte integer or not.

    Args:
        value (:obj:`str`): the value to be checked.

    Returns:
        :obj:`bool`: True if the value follows the format. False otherwise.
    """

    if not isinstance(value, str):
        return False
    else:
        if value.isdigit() and 0 <= int(value) <= 255:
            return True
        else:
            return False


def is_dotted_qad(value):
    """
    Checks whether a given value is in dotted quad format or nor:

    Dotted quad values are strings of four u1 integer separated by dotes:

    e.g. 127.0.0.1

    Args:
        value (:obj:`str`): the value to be checked.

    Returns:
        :obj:`bool`: True if the value is in dotted quad format.
            False otherwise.
    """

    if not isinstance(value, str):
        return False
    else:
        digits = value.split('.')
        if len(digits) != 4:
            return False
        else:
            if "/" in digits[3]:
                # Remove the mask from the IP (/value)
                digits[3] = digits[3][:digits[3].index('/')]
            return all(map(is_u1_string, digits))

def get_max_cases(df):
    """

def plot_funs(xs, funs, legs, colors, stys, ylog=False,
              title='Function growth', xlabel='$n$', ylabel='$f(n)$'):

    fig = plt.figure()

    for (f, col, sty) in zip(funs, colors, stys):
        ys = [f(x) for x in xs]
        plt.plot(xs, ys, sty, color=col)

    if ylog:
        plt.yscale("log")

    plt.legend(legs)
    plt.xlabel(xlabel)
    plt.ylabel(ylabel)
    plt.title(title)
    plt.show()

def get_min_value(df, col):
    """
    Return the minimum value found in column `col`
        of the dataframe `df`.
    """
    if is_string_dtype(df[col]):
        # Si la columna es de tipo cadena de caracteres, la convertimos
        # a datetime antes de calcular el mnimo
        i = np.argmin(np.array(pd.to_datetime(df[col])))
        return df[col][i]
    else:
        return min(df[col])

        # a datetime antes de calcular el mximo
        i = np.argmax(np.array(pd.to_datetime(df[col])))
        return df[col][i]
    else:
        return max(df[col])
